   2. Column-Specific Search: ប្រព័ន្ធនឹងស្វែងរកតែក្នុងជួរឈរ Primary Key (ដូចជា Order ID) ប៉ុណ្ណោះ ដែលកាត់បន្ថយបន្ទុករបស់ Google Server។

       * ខ្ញុំបានកែលម្អមុខងារភ្ជាប់ទៅកាន់ WebSocket ដោយបន្ថែមប្រព័ន្ធ Exponential Backoff សម្រាប់ការតភ្ជាប់ឡើងវិញ (Reconnect)។ ប្រសិនបើ Connection ដាច់
         វាអាចព្យាយាមតភ្ជាប់ដោយស្វ័យប្រវត្តិដោយពន្យារពេលបន្តិចម្តងៗ (ឧទាហរណ៍៖ 1s, 1.5s, 2.25s...) រហូតដល់អតិបរមា ៣០ វិនាទី ដើម្បីកុំឱ្យ Server ធ្វើការធ្ងន់ពេកនៅពេលមានបញ្ហាបណ្តាញអុីនធឺណិត។
       * បានបោសសម្អាត (Cleanup) Connection យ៉ាងត្រឹមត្រូវនៅពេល Component Unmount ដើម្បីជៀសវាងការខាតបង់ Memory (Memory Leaks)។

  ២. Optimized Data Syncing (Optimistic UI Updates):
       * នៅពេលទទួលបានសារ Update (update_order) ពី WebSocket, ប្រព័ន្ធឥឡូវនេះធ្វើការ Update State នៃបញ្ជីការកម្មង់ (orders) ដោយផ្ទាល់ភ្លាមៗ (Optimistic Update) ដោយមិនចាំបាច់ហៅ API
         ទាញយកបញ្ជីទាំងមូលឡើងវិញឡើយ។ នេះធ្វើឱ្យកម្មវិធីដើរលឿនជាងមុន និងមិនមានការកន្ត្រាក់។
       * ប្រសិនបើមានការ Update ថ្មី ប៉ុន្តែការកម្មង់នោះមិនទាន់មានក្នុងបញ្ជី (Local State) នោះទេ វានឹងទាញយក (Fetch) ទិន្នន័យឡើងវិញដោយស្វ័យប្រវត្តិ (Background Fetch) ដើម្បីធានាថាទិន្នន័យគឺថ្មីជានិច្ច។
       * បានបន្ថែមការចាប់យក Event sheet_webhook_sync ដើម្បីឱ្យកម្មវិធីអាច Auto-Refresh ប្រសិនបើមានអ្នកចូលទៅកែប្រែទិន្នន័យដោយផ្ទាល់នៅក្នុង Google Sheets។

  ៣. Cleaned up React Dependencies:
       * បានត្រួតពិនិត្យ និងរៀបចំ dependency array នៅក្នុង useMemo សម្រាប់ legacyContextValue ឱ្យបានត្រឹមត្រូវ រួមទាំងការបន្ថែម setOrders ទៅក្នុង Context Provider ផងដែរ ការពារមិនឱ្យមាន
         Stale Closures (ការចាប់យកតម្លៃចាស់របស់ State)។

  ៤. Build Check:
       * ខ្ញុំបានធ្វើការ Run Build បន្ទាប់ពីកែប្រែ ដើម្បីប្រាកដថាកូដមិនមាន Error ណាអាចរារាំងការ Build ឡើយ (Build ជោគជ័យ)។ 

  រាល់មុខងារនៃការតភ្ជាប់ទិន្នន័យ (Real-time features) នឹងដំណើរការបានយ៉ាងប្រសើរ ជាពិសេសសម្រាប់បុគ្គលិកវេចខ្ចប់ដែលត្រូវការភាពរហ័ស។

  ១. ផ្នែក Frontend (កន្លែងដែល Packer បញ្ចូលទិន្នន័យ)
  កូដសំខាន់បំផុតគឺនៅក្នុងឯកសារ components/admin/packaging/fastpack/FastPackTerminal.tsx ត្រង់មុខងារ executeFinalSubmit៖

   * ការប្រមូលទិន្នន័យ: វារៀបចំ Object មួយឈ្មោះ newData ដែលមាន៖
       * Fulfillment Status: ប្តូរទៅជា 'Ready to Ship'
       * Packed By: ឈ្មោះបុគ្គលិកដែលកំពុងប្រើប្រាស់កម្មវិធី
       * Packed Time: ពេលវេលាដែលចុចបញ្ចប់ការវេចខ្ចប់
   * ការផ្ញើទិន្នន័យ: 
       * ប្រសិនបើមានថតរូប៖ វានឹងហៅ API /api/upload-image ដើម្បីបង្ហោះរូបទៅ Google Drive ផង និង Update ទិន្នន័យទៅ Database ផង។
       * ប្រសិនបើគ្មានរូប (Manual Update)៖ វានឹងហៅ API /api/admin/update-order។

  ២. ផ្នែក Backend (កន្លែងរក្សាទុកក្នុង Database)
  កូដដែលទទួលការស្នើសុំពី Frontend គឺនៅក្នុង main.go ត្រង់មុខងារ handleAdminUpdateOrder៖

   * Validation: វាឆែកមើលថា តើការផ្លាស់ប្តូរ Status ត្រឹមត្រូវតាមលក្ខខណ្ឌដែរឬទេ (ឧទាហរណ៍៖ ពី Pending ទៅ Ready to Ship)។
   * Database Update: វាប្រើ GORM ដើម្បីរក្សាទុកទិន្នន័យចូលទៅក្នុងតារាង orders ក្នុង PostgreSQL។
   * Real-time Broadcast: បន្ទាប់ពី Save រួច វានឹងបញ្ជូនសារតាមរយៈ WebSocket (hub.Broadcast) ទៅកាន់ User ផ្សេងទៀតឱ្យដឹងថាមានការ Update។
   * Sync to Sheets: វានឹងហៅមុខងារ enqueueSync ដើម្បីដាក់ទិន្នន័យចូលក្នុង Queue សម្រាប់ផ្ញើទៅ Google Sheets។

  ៣. ផ្នែក Google Apps Script (កន្លែង Sync ទៅ Google Sheets & Telegram)
  នៅក្នុងឯកសារ App Script/code_gs_api.gs មានមុខងារសំខាន់ៗ៖

   * handleUpdateSheet: ទទួលខុសត្រូវក្នុងការស្វែងរកជួរ (Row) ក្នុង Google Sheet តាមរយៈ Order ID ហើយកែប្រែព័ត៌មានក្នុងតារាង AllOrders និងតារាង Team ជាក់ស្តែង។
   * updateOrderTelegram: បន្ទាប់ពីកែក្នុង Sheet រួច វានឹងកែប្រែសារ (Edit Message) នៅក្នុង Group Telegram ដើម្បីឱ្យអ្នកដឹកជញ្ជូន (Driver) ឃើញថាអីវ៉ាន់នេះ "វេចខ្ចប់រួចរាល់"។

  សង្ខេបវាលទិន្នន័យ (Fields) ដែលពាក់ព័ន្ធក្នុង Database:
  នៅក្នុង Backend/models.go ជួរឈរដែលរក្សាទុកទិន្នន័យ Packing មាន៖
   1. fulfillment_status (ស្ថានភាព: Ready to Ship)
   2. packed_by (ឈ្មោះអ្នកវេចខ្ចប់)
   3. packed_time (ម៉ោងវេចខ្ចប់)
   4. package_photo_url (Link រូបភាពភស្តុតាងនៃការវេចខ្ចប់)

        Script យល់ និងដាក់ក្នុង Sheet បានត្រឹមត្រូវ។

  ប្រសិនបើទិន្នន័យក្នុង Google Sheet របស់អ្នកមានបញ្ហា Case-Sensitive (ដូចជា "Sales" ទល់នឹង "sales") ឬប្រសិនបើមាន Column ណាដែលរកមិនឃើញតាមរយៈ normalizeKey វានឹងធ្វើឱ្យ rowIndex នៅតែស្មើ -1 (រកមិនឃើញ) ដូចនេះវាមិនធ្វើការ Update អ្វីទាំងអស់។
