តាមរយៈរចនាសម្ព័ន្ធកូដនៃគម្រោងរបស់អ្នក ការ Compression រូបភាពគឺត្រូវបានធ្វើឡើងនៅ ម៉ាស៊ីនរបស់ User (Client-side) ជាចម្បង មុនពេលរូបភាពនោះត្រូវបានផ្ញើទៅកាន់ Server។

  នេះគឺជាមូលហេតុ និងភស្តុតាងបច្ចេកទេស៖

   1. មាន File utils/imageCompressor.ts: នេះគឺជា TypeScript utility ដែលដំណើរការនៅលើ Browser របស់អ្នកប្រើប្រាស់។ វាប្រើប្រាស់បច្ចេកវិទ្យាដូចជា canvas ដើម្បីកាត់បន្ថយទំហំរូបភាព (Resize/Compress) ភ្លាមៗបន្ទាប់ពី User
      រើសរូបភាពរួច។
   2. អត្ថប្រយោជន៍: ការធ្វើបែបនេះគឺដើម្បី៖
       * កាត់បន្ថយ Bandwidth: បើថតរូបចេញពី iPhone/Android អាចមានទំហំ 5MB-10MB បើយើង Compress នៅលើម៉ាស៊ីន User ឱ្យសល់ត្រឹម 200KB-500KB នោះការ Upload នឹងលឿនខ្លាំង។
       * កាត់បន្ថយបន្ទុក Server: Server មិនចាំបាច់ចំណាយកម្លាំង CPU មកអង្គុយ Compress រូបភាពរាប់ពាន់សន្លឹកក្នុងពេលតែមួយនោះទេ។

  សរុបមក៖ ម៉ាស៊ីនរបស់ User (Browser/Phone) ជាអ្នករ៉ាប់រងការ Compress រូបភាព មុននឹងបោះទៅឱ្យ Server រក្សាទុកក្នុង Database ឬ Cloud។





  ខ្ញុំបានកែលម្អ QR Code នៅលើ Label ឱ្យកាន់តែមានប្រសិទ្ធភាព និងមានសោភ័ណភាពខ្ពស់ដូចខាងក្រោម៖

   1. ពង្រឹងសមត្ថភាពស្កេន (High-Reliability Scanning):
       * កំណត់កម្រិត Error Correction ទៅជា "High" (H)៖ ទោះបីជា Label រហែក ឬប្រឡាក់បន្តិចបន្តួច ក៏នៅតែអាចស្កេនបានយ៉ាងងាយស្រួល។
       * បន្ថែម Quiet Zone (ចន្លោះសជុំវិញ QR)៖ ដើម្បីឱ្យកាមេរ៉ាទូរស័ព្ទ ឬម៉ាស៊ីនស្កេនចាប់យកទិន្នន័យបានលឿនជាងមុន។

   2. ការរចនាបែបអាជីព (Professional Branding):
       * បន្ថែម Logo (Box Icon) នៅចំកណ្តាល QR Code ដើម្បីឱ្យមើលទៅមានលក្ខណៈ Premium និងសម្គាល់ម៉ាកយីហោបានច្បាស់។
       * ប្រើប្រាស់ជ្រុងមូល (Rounded Corners) សម្រាប់ Frame របស់ QR ដើម្បីឱ្យស៊ីគ្នាជាមួយ Design ទំនើបនៃ Label ទាំងមូល។

   3. រចនាបថ High-tech នៅលើ Label:
       * ក្នុង FlexiLabel ខ្ញុំបានបន្ថែម Tech Frame Indicators (ស៊ុមជ្រុងតូចៗ) នៅជុំវិញ QR Code ដែលធ្វើឱ្យវាមានលក្ខណៈដូចជាប្រព័ន្ធ Logistics ទំនើប។
       * កែលម្អទំហំឱ្យមានតុល្យភាពល្អជាមួយព័ត៌មានផ្សេងៗនៅលើ Label។
