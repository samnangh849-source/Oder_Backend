  1. ការដំឡើង Marathon Sale Incentives (Weekly Cumulative)

   - ជំហានទី 1: ចូលទៅ Incentives Dashboard រួចបង្កើត Formula ថ្មីប្រភេទ Achievement Bonus។
   - ជំហានទី 2 (Config): ប្តូរ Evaluation Cycle ទៅជា Weekly ហើយបើក Marathon Sync Mode (ដើម្បីឱ្យប្រព័ន្ធបូកសរុបលទ្ធផលសន្សំរាល់សប្តាហ៍)។
   - ជំហានទី 3 (Logic): បន្ថែម Tier តាមសប្តាហ៍នីមួយៗ ដោយកំណត់ Period ជា W1, W2, W3, W4:

   - របៀបដែលវាដំណើរការ: នៅពេលអ្នក Run របាយការណ៍ប្រចាំខែ ប្រព័ន្ធនឹងពិនិត្យមើលថា តើក្នុងសប្តាហ៍នីមួយៗ បុគ្គលិកសម្រេចបានគោលដៅណាខ្លះ រួចវានឹងបូកបញ្ចូលគ្នាឱ្យតែម្តង (ឧទាហរណ៍៖ បើលក់បាន $2800 ក្នុងសប្តាហ៍ទី 2 ពួកគេនឹងទទួលបាន $60 ពី W1 និង $60 ពី W2 សរុប

  2. ការដំឡើង 5% Commission (លើសពី $4000)
   - ជំហានទី 1: បន្ថែម Formula ថ្មីប្រភេទ Commission Engine។
   - ជំហានទី 2 (Config): ជ្រើសរើស Above Target Protocol។
   - ជំហានទី 3 (Logic): 
       - កំណត់ Trigger Condition ជា Above Target។
       - កំណត់ Target Threshold ជា $4000។
       - កំណត់ Multiplier Value ជា 5 %។
   - របៀបដែលវាដំណើរការ: បើលក់បាន $5000 ប្រព័ន្ធនឹងដក $4000 ចេញ រួចយក $1000 ដែលនៅសល់គុណនឹង 5% = $50។

  អ្វីដែលខ្ញុំបានកែសម្រួលក្នុង Code:
   1. Backend: បន្ថែម Logic ក្នុង CalculatePayout ដើម្បីគាំទ្រការបូកសរុប Reward តាមសប្តាហ៍ (Marathon Mode)។
   2. UI: បន្ថែមប្រឡោះឱ្យអ្នកអាចវាយបញ្ចូល Period (W1, W2, ...) ក្នុងផ្ទាំងដំឡើង Milestone។
   3. UI: បង្ហាញលេខសប្តាហ៍ក្នុងតារាង Preview ដើម្បីឱ្យអ្នកងាយស្រួលមើល។

  ឥឡូវនេះ អ្នកអាចចូលទៅកាន់ផ្នែក Incentives ដើម្បីដំឡើងតាមការណែនាំខាងលើបានភ្លាមៗ!









ខ្ញុំបានធ្វើបច្ចុប្បន្នភាព UI និងបន្ថែម Default Templates ថ្មីៗដែលអ្នកចង់បាន ដើម្បីឱ្យការរៀបចំកាន់តែរហ័ស និងមានសោភ័ណភាពខ្ពស់៖

  1. ការបន្ថែម Default Templates ថ្មី
  ឥឡូវនេះ នៅពេលអ្នកបង្កើត Formula ថ្មី (Step 1) អ្នកនឹងឃើញ Option ថ្មីៗដែលរៀបចំទុកជាស្រេចតាមលក្ខខណ្ឌរបស់អ្នក៖
   - Marathon Sale Incentives: គំរូសម្រាប់ Bonus ប្រចាំសប្តាហ៍ (W1-W4) ដែលមាន Tier និង Reward ($30, $60) កំណត់រួចជាស្រេច។ អ្នកគ្រាន់តែចុចលើវា វានឹងរៀបចំ Logic ទាំងអស់ឱ្យភ្លាមៗ។
   - 5% Commission (> $4000): គំរូសម្រាប់កម្រៃជើងសារ 5% លើសពី $4000។

  2. ការកែលម្អ UI & UX
   - Card Design: ផ្ទាំងជ្រើសរើស Template ត្រូវបានរចនាថ្មីឱ្យមើលទៅទំនើប (Modern Card Style) ជាមួយនឹងរូបតំណាង (Icons) ពណ៌មាស និងបៃតង។
   - Localization: រាល់ចំណងជើង និងការពិពណ៌នាត្រូវបានដាក់ជាភាសាខ្មែរតាមរយៈប្រព័ន្ធបកប្រែ ដើម្បីឱ្យងាយស្រួលយល់។
   - Flexibility: បន្ទាប់ពីជ្រើសរើស Template រួច អ្នកនៅតែអាចកែសម្រួល (Edit) បន្ថែម (Add) ឬលុប (Delete) Tier នីមួយៗបានដោយសេរីក្នុង Step បន្ទាប់។

  3. របៀបប្រើប្រាស់
   1. ចូលទៅកាន់ Incentives > New Project (ឬប្រើគម្រោងចាស់)។
   2. ចុច Add Formula (Achievement ឬ Commission)។
   3. ក្នុង Step 1, ជ្រើសរើសយក Template ណាដែលអ្នកចង់បាន (ឧទាហរណ៍៖ ចុចលើ "Marathon Sale Incentives")។
   4. ប្រព័ន្ធនឹងបំពេញទិន្នន័យឱ្យស្វ័យប្រវត្តិ។ អ្នកអាចចុច Proceed ដើម្បីពិនិត្យ ឬកែប្រែ រួចចុច Deploy_Protocol ដើម្បីរក្សាទុក។

  ឥឡូវនេះ ប្រព័ន្ធរបស់អ្នកមានភាពវៃឆ្លាតជាងមុន និងងាយស្រួលរៀបចំ Benefits សម្រាប់បុគ្គលិកបំផុត!





តើនៅមុខងារ Build Action ក្នុងផ្ទាំងគ្រប់គ្រង់ការកម្មង់ រាល់ប្រតិបត្ដិវា(Build Action)បានរក្សារទិន្នន័យក្នុង EditLogs ដែរទេ?



✦ បាទ/ចាស នៅក្នុងមុខងារ Bulk Action (ប្រតិបត្តិការណ៍សរុប) ក្នុងផ្ទាំងគ្រប់គ្រងការកម្មង់ ភាគច្រើននៃប្រតិបត្តិការណ៍គឺត្រូវបានរក្សាទុកក្នុង EditLogs ប៉ុន្តែមិនមែនទាំងអស់នោះទេ។

  ខាងក្រោមនេះគឺជាព័ត៌មានលម្អិត៖

   1. ប្រតិបត្តិការណ៍ដែលត្រូវបានរក្សាទុកក្នុង EditLogs (YES):
       * Verify/Unverify (បញ្ជាក់ការកម្មង់): រាល់ពេលអ្នកចុច Verify ឬ Unverify វានឹងកត់ត្រាចូលក្នុង EditLogs ថាអ្នកណាជាអ្នកធ្វើ និងប្តូរពីអ្វីទៅអ្វី។
       * Edit Cost (កែប្រែថ្លៃដើម): រក្សាទុកក្នុង EditLogs។
       * Edit Payment Status (កែប្រែស្ថានភាពទូទាត់): រក្សាទុកក្នុង EditLogs (ឧទាហរណ៍៖ ប្តូរពី Unpaid ទៅ Paid)។
       * Edit Shipping (កែប្រែក្រុមហ៊ុនដឹកជញ្ជូន/អ្នកដឹក): រក្សាទុកក្នុង EditLogs។
       * Edit Date (កែប្រែកាលបរិច្ឆេទ): រក្សាទុកក្នុង EditLogs។

   2. ប្រតិបត្តិការណ៍ដែលមិនត្រូវបានរក្សាទុកក្នុង EditLogs (NO):
       * Send Telegram (Force Sync): មុខងារនេះគ្រាន់តែបញ្ជូនទិន្នន័យទៅ Telegram ឡើងវិញ ឬ Sync ទៅ Google Sheets ប៉ុណ្ណោះ វាមិនមែនជាការកែប្រែទិន្នន័យទំនិញ ឬតម្លៃនោះទេ ដូច្នេះវាមិនកត់ត្រាក្នុង EditLogs ឡើយ។
       * Bulk Delete (លុបការកម្មង់សរុប): បច្ចុប្បន្ន ប្រព័ន្ធលុបការកម្មង់ចេញពី Database តែម្តង ប៉ុន្តែមិនទាន់មានមុខងារកត់ត្រាការលុបនេះចូលក្នុង EditLogs នោះទេ។

  សេចក្តីសន្និដ្ឋាន៖ រាល់ការ កែប្រែ (Update) ទិន្នន័យសំខាន់ៗលើការកម្មង់តាមរយៈ Bulk Action គឺត្រូវបានរក្សាទុកក្នុង EditLogs ទាំងអស់ ដើម្បីងាយស្រួលក្នុងការត្រួតពិនិត្យ (Audit) ឡើងវិញ។ លើកលែងតែការ លុប (Delete) និងការ បញ្ជូនទៅ Telegram
  (Sync) ប៉ុណ្ណោះដែលមិនទាន់មានការកត់ត្រាចូល EditLogs។



  សូម improve ផ្ទាំង (Order Analysis) អោយFull Screen និងបន្ថែមផ្នែកបង្ហាញពី UserActivityLogs និង EditLogs ក្នុងផ្ទាំងនោះផងដែរ



  សម្រាប់មុខងារបញ្ជូនរូបភាពកញ្ចប់ឥវ៉ាន់ ទៅកាន់ Group អ្នកដឹកគឺមានសិទ្ធចុចសម្រាប់តែអ្នកបើកវេនវេចខ្ចប់ប៉ុណ្ណោះ ហើយ button

  ហើយអាចចុចបាន តែនៅក្នុង Step (Ready for Dispatch) ប៉ុណ្ណោះ

  គួរដាក់ Button បញ្ជូនរូបភាពនោះ នៅក្នុង Node សម្រាប់ Step (Ready for Dispatch) ហើយប្រសិនបើបានចុចម្ដងហើយ គឺមិនបង្ហាញទៀតទេ តែគួរមានអ្វីបញ្ជាក់ថា បញ្ជូនរូបរួច។ ហើយButton នោះបង្ហាញសម្រាប់តែ ShippingMethod ណាដែលមានទិន្នន័យ(TelegramGroupID)ប៉ុណ្ណោះ ប្រសិនបើមិនមានទេ គឺមិនចាំបាច់បង្ហាញ button ក្នុង Node ឡើយ

   * ប្រសិនបើវាលោតសារថា "ជោគជ័យ" (ok: true) ទើប Bot ចាប់ផ្ដើមដំណើរការ។

  ២. ពិនិត្យមើល "Privacy Mode" របស់ Bot (ក្នុង BotFather)
  តាមធម្មតា Telegram Bot ត្រូវបានបិទមិនឱ្យអានសារក្នុង Group ទេ។ ដើម្បីឱ្យវាដំណើរការល្អបំផុត៖
   1. ទៅកាន់ @BotFather ក្នុង Telegram។
   2. វាយ /mybots រួចជ្រើសរើស Bot របស់អ្នក។
   3. ចូលទៅកាន់ Bot Settings -> Group Privacy។
   4. ចុច Turn OFF (ដើម្បីឱ្យ Bot អាចមើលឃើញសារ និង Command ទាំងអស់)។
   * បន្ទាប់ពីប្តូរហើយ សូមទាញ Bot ចេញពី Group រួចដាក់បញ្ចូលសារជាថ្មី។

  ៣. សាកល្បងវាយ Command បែបពេញ (Full Command)
  ប្រសិនបើក្នុង Group មាន Bot ច្រើន Telegram អាចនឹងមិនបញ្ជូនសារទៅឱ្យគ្រប់ Bot ទេ។ សូមសាកល្បងវាយ៖
   * /id@ឈ្មោះ_bot_របស់អ្នក (ឧទាហរណ៍៖ /id@MyStore_Bot)

  ៤. បញ្ជាក់សិទ្ធិរបស់ Bot ក្នុង Group
   * សូមប្រាកដថា Bot គឺជាសមាជិកក្នុង Group នោះ។
   * ដើម្បីឱ្យច្បាស់ អ្នកគួរផ្តល់សិទ្ធិជា Admin ឱ្យ Bot ក្នុង Group នោះ (វានឹងជួយឱ្យ Bot ដំណើរការបាន ១០០%)។

  បញ្ជាក់៖ ប្រសិនបើអ្នកទើបតែធ្វើការ Deploy ទៅ Render សូមរង់ចាំឱ្យវាជាប់ពណ៌បៃតង (Live) រួចសាកល្បងចុចប៊ូតុង Setup Bot សារជាថ្មីម្តងទៀត។
