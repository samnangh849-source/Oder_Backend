`សូមអភិវឌ្ឍ និងរកកំហុស នៃផ្នែកPermissions Managementក្នុង(view=admin_dashboard&tab=settings) ដោយត្រូវពិនិត្យទាំង Backend និង Frontend រួមទាំង App Script ផងដែរ`


១. ការដោះស្រាយបញ្ហា Build Failure លើ Render:
   * បានធ្វើបច្ចុប្បន្នភាពឯកសារ go.mod និង go.sum ដោយប្រើបញ្ជា go get និង go mod tidy ដើម្បីបញ្ចូល Dependencies ដែលបាត់ (ដូចជា gin, gorm, jwt, google-api)។
     នេះនឹងដោះស្រាយបញ្ហា "missing go.sum entry" ដែលធ្វើឱ្យការ Deploy បរាជ័យ។

  ២. ការកែលម្អផ្នែក Permissions Management (Frontend):
   * មុខងារស្វែងរក (Feature Search): បន្ថែមរបារស្វែងរកក្នុងតារាង Permissions ដើម្បីងាយស្រួលរកមុខងារនីមួយៗ។
   * ការចុះឈ្មោះ Role (Orphan Role Registration): បន្ថែមប៊ូតុង "Register" សម្រាប់ Role ណាដែលមានប្រើក្នុងតារាង Users ប៉ុន្តែមិនទាន់មានក្នុងតារាង Roles ជាផ្លូវការ។
   * Optimistic UI: កែសម្រួលឱ្យការ Toggle ប្តូរសិទ្ធិមានភាពរលូន និងមិនទាក់ (no flicker)។

  ៣. ការបង្កើនល្បឿន និងប្រសិទ្ធភាព (Backend & App Script):
   * Batch Sync: កែសម្រួលការ Sync Permissions ទៅ Google Sheet ឱ្យប្រើប្រាស់ batchAddRows (បញ្ជូនទិន្នន័យទាំងអស់ក្នុងពេលតែមួយ) ជំនួសឱ្យការបញ្ជូនម្តងមួយជួរ ដែលធ្វើឱ្យការ
     Sync លឿនជាងមុន ១០ដង។
   * Safe Reset: កែសម្រួល HandleResetPermissions ឱ្យលុបតែសិទ្ធិរបស់ Role ស្តង់ដារ (Admin, Sales...) និងរក្សាសិទ្ធិរបស់ Role ថ្មីៗដែល Admin បង្កើតដោយខ្លួនឯង។

  ៤. App Script:
   * បន្ថែម Action batchAddRows ក្នុង code_gs_api.gs ដើម្បីគាំទ្រការបញ្ចូលទិន្នន័យជាដុំធំ (Bulk Insert)។

  ចំណាំ: សូមធ្វើការ Push ការផ្លាស់ប្តូរទាំងនេះទៅកាន់ GitHub ដើម្បីឱ្យ Render ធ្វើការ Build សារជាថ្មី។
